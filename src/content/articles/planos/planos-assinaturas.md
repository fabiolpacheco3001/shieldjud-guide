# Planos e assinaturas

O ShieldJud funciona por meio de **planos de assinatura**, que definem o volume de uso da plataforma e a forma de organização dos usuários.

Os planos foram estruturados para atender diferentes perfis de utilização, desde usuários individuais até equipes ou organizações com maior volume de processos.

A principal diferença entre os planos está relacionada a:

- quantidade de **transações mensais**
- capacidade da **biblioteca de documentos**
- forma de **gestão de usuários**
- recursos de **integração e suporte**

---

# Estrutura geral dos planos

Atualmente o ShieldJud possui quatro categorias de plano:

- **Plano Free**
- **Planos Individuais**
- **Planos de Gabinete**
- **Plano Enterprise**

Cada categoria foi pensada para um nível diferente de uso da plataforma.

---

# O que é considerado uma transação

No ShieldJud, cada **transação** corresponde ao processamento de um processo enviado para análise.

Uma transação inclui:

- **1 petição inicial**
- até **40 documentos anexos**

Cada transação gera os artefatos de análise disponíveis no sistema.

---

# Visualização e histórico das análises

Todos os planos oferecem acesso contínuo às transações já realizadas.

Isso significa que o usuário possui:

- **visualização ilimitada** das análises já processadas
- **histórico permanente** das transações realizadas
- **interação conversacional ilimitada** dentro das transações existentes

Mesmo após atingir o limite mensal de transações, o usuário continua podendo consultar e interagir com os processos já analisados.

---

# Biblioteca de documentos

Todos os planos incluem acesso à **biblioteca de documentos**.

A biblioteca permite armazenar materiais que podem servir como referência ou contexto adicional para as análises realizadas no sistema.

Entre os exemplos de uso da biblioteca estão:

- precedentes relevantes
- notas técnicas
- orientações institucionais
- documentos jurídicos de referência

Cada documento da biblioteca possui limite de tamanho de **até 200 KB**.

A quantidade de documentos permitidos varia de acordo com o plano contratado.

---

# Planos individuais

Os **Planos Individuais** são voltados para uso por uma única pessoa.

Nesse modelo:

- cada usuário possui **login próprio**
- a biblioteca é **associada ao usuário**
- o volume de transações varia conforme o plano escolhido

Esses planos são adequados para:

- magistrados que utilizam o sistema individualmente
- assessores jurídicos
- usuários que analisam processos de forma recorrente

---

# Planos de gabinete

Os **Planos de Gabinete** foram projetados para uso compartilhado por pequenas equipes.

Nesse modelo:

- existe um **gestor do gabinete**
- podem existir **até três usuários adicionais**
- cada usuário possui **login próprio**

O gestor possui acesso a todas as transações realizadas pelos membros do gabinete.

Além disso, a biblioteca é **compartilhada entre os usuários do gabinete**, permitindo centralizar documentos de referência da unidade.

O gestor também pode:

- incluir novos usuários
- remover usuários
- delegar a gestão da biblioteca

---

# Plano Enterprise

O **Plano Enterprise** é destinado a organizações que precisam de uso mais amplo da plataforma.

Esse plano inclui recursos adicionais voltados à operação institucional, como:

- **transações ilimitadas**
- gestão de múltiplos gabinetes
- biblioteca ampliada
- **integração com sistemas próprios**
- **API dedicada**
- **SLA e suporte prioritário**

Esse modelo é indicado para ambientes que demandam uso intensivo e integração com infraestrutura tecnológica existente.

---

# Alteração de plano

O usuário pode alterar seu plano sempre que necessário.

Dependendo da mudança solicitada, a alteração pode ocorrer:

- imediatamente
- no próximo ciclo de assinatura

As alterações podem incluir:

- upgrade para plano superior
- migração para plano mais simples
- mudança entre categorias de plano

---

# Cancelamento da assinatura

A assinatura pode ser cancelada a qualquer momento pelo usuário.

Após o cancelamento:

- o acesso permanece ativo até o final do ciclo vigente
- novas cobranças não são realizadas

Após o término do período contratado, o acesso aos recursos da plataforma é encerrado.
